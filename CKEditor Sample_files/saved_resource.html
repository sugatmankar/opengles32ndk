      3. Save Project Location
      4. Language
      5. Minimum level as per your requirement

    Make sure everything is successfully done.

    5. Deploy apk to Mobile

    1. IDE way :
      1. Build Menu - > Make Project
      2. Run ->     Run App -> Select Deployment Target - > Connected Devices -> OK
    2. Command Line way:
      1. Open CMD and cd to project root directory
      2. gradlew clean
      3. gradlew build
      4. adb -d install -r app/build/outputs/apk/debug/app-debug.apk


    3. Understanding Build System

    Android Developer Guide has provided plenty of information about entire build system, how does android build system works, have a look at developers guide. Here i'm just explaining wrt NDK's perspective. In your project structure you will find two build.gradle files, one is in project root directory and another is in project/app/ directory. For compilation of C C++ native code android has used CMake build system, hence you will find externalNativeBuild block in app/build.gradle file with path of CMake file, this is very important for us to add classes we have/will written/write in C CPP as well as when we will add external libraries like freetype for font rendering.

    By default project structure generated one cpp folder under that you will find includes , CMakelist.txt and native-lib.cpp file

    Lets understand flow of default program and later we will add some more example

    In MainActivity -> tv.setText(StringFromJNI()) stringfromjni function will be called as it is native mark it will look for function definition in cpp file 

    now control is in cpp files implementation of StringFromJNI method

    Lets write some more basic fundamental data type return from cpp to java

    Just remember basic types in jni

    ```cpp

    typedef long               jint;
    typedef __int64         jlong;
    typedef signed char     jbyte;
    typedef unsigned char   jboolean;
    typedef unsigned short  jchar;
    typedef short           jshort;
    typedef float           jfloat;
    typedef double          jdouble;
    typedef jint            jsize;
    ```

    Write methods which return int long byte etc...

    Implement those methods in native-lib.cpp as shown below

    Things need to take care

    In Java file your method declaration must be in following syntax

    ```java
    public native ReturnType methodName(Parameters if any);
    ```

    In CPP file your declared method must begin with syntax Considering you java class (MainActivity in my case) is located in package com.sugat.hellondk , this class has native declared methods

     ```cpp

    extern "C" JNIEXPORT RETURNTYPE_JNI JNICALL

    Java_com_sugat_hellondk_MainActivity_MethodName

    ```

    Lets try for two parameter method

    ```java

    public native int sumOfTwoInt(int i,int j);
    ```

    ```cpp

    Java_com_sugat_hellondk_MainActivity_sumOfTwoInt(JNIEnv *env, jobject instance, jint i, jint j)
    ```
    First two parameters are must JNIENV and jobject. 
    1. JNIENV *env is for accessing JNI environment interface
    2. jobject instance is for referencing this object
